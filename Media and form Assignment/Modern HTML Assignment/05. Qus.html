Tab index

The tabindex global attribute allows developers to make HTML elements focusable, allow or prevent them from being sequentially focusable (usually with the Tab key, hence the name) and determine their relative ordering for sequential focus navigation.

  1. Purpose: The tab index attribute is used to specify the sequential order in which elements can be accessed using the keyboard's Tab key. By default, elements such as links, form inputs, and buttons are included in the natural tab order based on their position in the HTML source code. However, the tab index attribute allows developers to override this default order and customize the focus navigation.
  2. Values: The tab index attribute accepts numerical values, where each value represents the order of the element in the tab sequence. Elements with a lower tab index value will be focused first when navigating using the Tab key. It is important to note that only elements with a positive tab index value (greater than or equal to 1) are included in the tab order. Elements with a tab index of 0 are also considered part of the tab order but are focused last, after elements with positive tab index values.
  3. Skipping elements: In some cases, developers may want to exclude certain elements from the tab order. To achieve this, they can set the tab index attribute to a negative value (-1). Elements with a tab index of -1 are programmatically focusable but are skipped when navigating using the Tab key. This is useful for elements that should not receive keyboard focus, such as decorative images or hidden elements that are accessed via other means.
